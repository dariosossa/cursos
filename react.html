4 Aprendiendo DOM

El DOM (Document Object Model) es un modelo estándar que representa la estructura de un documento HTML como un árbol de nodos. Cada etiqueta, atributo y fragmento de texto se convierte en un objeto que puede ser accedido y manipulado mediante JavaScript, permitiendo que una página web sea dinámica e interactiva.

Gracias al DOM, el desarrollador puede leer, modificar, crear o eliminar elementos de la interfaz en tiempo de ejecución. Esto hace posible responder a acciones del usuario como clics, escritura en formularios o desplazamientos, actualizando el contenido y la apariencia de la página sin necesidad de recargarla.

Método Tradicional vs Moderno

El Método Tradicional, consiste en usar getElementById, mientras que el Método Moderno, consiste en usar querySelector.

¿Cómo se trabaja con 'getElementById'?

En realidad getElementById es un método del objeto document que permite capturar un elemento HTML usando su atributo id. Es una de las formas más básicas, rápidas y clásicas de acceder al DOM con JavaScript.

Se captura el elemento de la siguiente manera:

DOM clásico
// Captura clásica por ID (sin #) const caja = document.getElementById("caja");

¿Cómo se trabaja con 'querySelector'?

En realidad querySelector es un método moderno del objeto document que permite seleccionar el primer elemento del DOM que coincida con un selector CSS válido. A diferencia de getElementById, sí utiliza la sintaxis de los selectores CSS.

Se captura el elemento de la siguiente manera:

DOM moderno
// Usa selectores CSS const caja = document.querySelector('.caja');

¿Cómo se trabaja con 'querySelectorAll'?

En realidad querySelectorAll es un método del objeto document que permite seleccionar todos los elementos del DOM que coincidan con un selector CSS válido. A diferencia de querySelector, no devuelve un solo elemento, sino una colección de elementos.

Se capturan los elementos de la siguiente manera:

Colecciones
// Devuelve una NodeList const caja = document.querySelectorAll('.caja');

Clases CSS: Añadir y Quitar Estilos

En el DOM, la forma correcta de aplicar estilos no es modificando cada propiedad con style, sino utilizando clases CSS. JavaScript se encarga únicamente de agregar o quitar clases, y el navegador aplica automáticamente los estilos definidos en CSS.

Esta clase no se aplica sola. Solo tendrá efecto cuando JavaScript la agregue a un elemento del DOM. Cuando se ejecuta classList.add(), el navegador busca la clase en el CSS y aplica todos sus estilos al elemento.

Quitar clase

// Quita la clase CSS
caja.classList.remove('resaltado');

Al usar classList.remove(), el estilo desaparece y el elemento vuelve a su estado original.

Alternar

// Agrega o quita la clase automáticamente
caja.classList.toggle('resaltado');

toggle es muy usado para menús, botones, modos oscuro/claro y mostrar u ocultar elementos, porque decide automáticamente si agrega o quita la clase.

6. Carpetas y Archivos de React

node_modules: Esta carpeta contiene todas las dependencias del proyecto. Aquí se guardan las librerías que React y Vite necesitan para funcionar. No se crea manualmente y no se sube a GitHub, porque se puede reconstruir usando el comando npm install.


public: En esta carpeta se colocan archivos estáticos que no pasan por React, como imágenes, íconos o archivos que deben mantenerse con la misma ruta. Todo lo que esté aquí se puede acceder directamente desde el navegador.


src: Es la carpeta más importante del proyecto. Aquí se escribe todo el código de React: componentes, estilos, lógica y archivos JavaScript o JSX. Desde esta carpeta se construye toda la aplicación.


.gitignore: Este archivo indica qué carpetas o archivos no deben subirse a un repositorio Git. Normalmente se excluyen node_modules y otros archivos generados automáticamente.


eslint.config.js: Este archivo configura ESLint, una herramienta que ayuda a detectar errores y malas prácticas en el código. Sirve para mantener el código limpio y ordenado mientras se programa.


index.html: Es el archivo HTML principal del proyecto. React se monta dentro de este archivo. Aquí normalmente se encuentra un solo elemento div donde React renderiza toda la aplicación.


package.json: Contiene la información del proyecto, las dependencias que se usan y los comandos disponibles. Es fundamental para saber cómo se ejecuta y se construye la aplicación.


package-lock.json: Este archivo guarda la versión exacta de cada dependencia instalada. Sirve para que el proyecto funcione igual en todas las computadoras.


README.md: Es un archivo de documentación. Aquí se suele explicar qué hace el proyecto y cómo ejecutarlo. No afecta al funcionamiento de la aplicación.


vite.config.js: Archivo de configuración de Vite. Permite ajustar cómo se ejecuta el servidor de desarrollo y cómo se construye el proyecto para producción. Normalmente no se modifica al inicio.

7. INDEX.html de React

¿Qué es index.html?
El archivo index.html es el archivo HTML principal del proyecto. Es el primer archivo que el navegador carga cuando se ejecuta la aplicación. React no reemplaza este archivo, sino que se monta dentro de él.


Rol de index.html en React
En una aplicación React, index.html contiene una estructura mínima. Su función principal es proporcionar un punto donde React pueda renderizar toda la interfaz de la aplicación.


Código real de index.html por defecto
A continuación se muestra el contenido real del archivo index.html que viene por defecto al crear un proyecto con Vite + React.


index.html

<!DOCTYPE html>
<html lang="es">
  <head>
    <meta charset="UTF-8" />
    <link rel="icon" type="image/svg+xml" href="/vite.svg" />
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
    <title>React + Vite</title>
  </head>
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>

El contenedor root
Dentro del body existe un elemento div con el identificador root. Este elemento es fundamental, porque es el lugar donde React insertará toda la aplicación. Sin este contenedor, React no puede mostrarse en pantalla.


Conexión con main.jsx
El archivo index.html carga el archivo main.jsx mediante una etiqueta script de tipo módulo. Cuando el navegador ejecuta ese script, React se inicializa y renderiza el componente App dentro del elemento root.


Por qué index.html casi no se modifica
En React, casi todo el desarrollo ocurre dentro de la carpeta src. El archivo index.html suele mantenerse simple y estable, ya que solo sirve como base para que React funcione correctamente.


Relación general del flujo
Primero el navegador carga index.html. Luego se ejecuta main.jsx. Finalmente, React renderiza App.jsx y los demás componentes. Este flujo es la base de cualquier aplicación React.

8. Main de React

¿Qué es main.jsx?
El archivo main.jsx es el punto de entrada de la aplicación React. Es el primer archivo JavaScript que se ejecuta cuando se carga el proyecto en el navegador. Desde aquí se inicia React y se conecta la aplicación con el archivo index.html.


¿Para qué sirve main.jsx?
Su función principal es decirle a React en qué parte del HTML debe mostrarse la aplicación y cuál es el componente principal que se va a renderizar. Sin este archivo, React no sabría dónde ni qué mostrar.


¿Qué contiene main.jsx?
Este archivo normalmente importa React, el método de renderizado y el componente principal de la aplicación. También se encarga de enlazar React con el elemento HTML que actúa como contenedor.


main.jsx

import { StrictMode } from 'react'
import { createRoot } from 'react-dom/client'
import './index.css'
import App from './App.jsx'

ReactDOM.createRoot(document.getElementById('root')).render(
  <StrictMode>
    <App />
  </StrictMode>
)

Conexión con index.html
Dentro de index.html existe un elemento con un identificador, generalmente llamado root. El archivo main.jsx utiliza ese identificador para montar toda la aplicación React en ese lugar.

9. Componente React

¿Qué es un componente en React?
Un componente en React es una pieza de código reutilizable que representa una parte de la interfaz de usuario. Puede ser un botón, un formulario, un encabezado o una sección completa de la página. React construye la aplicación uniendo varios componentes.


Para qué sirven los componentes
Los componentes permiten dividir la interfaz en partes pequeñas y ordenadas. Esto hace que el código sea más fácil de leer, mantener y reutilizar. Si algo falla en un componente, solo se revisa ese archivo.


Cómo se crea un componente en React
Un componente se crea como una función de JavaScript que devuelve código JSX. Por convención, el nombre del componente debe comenzar con letra mayúscula. Cada componente se guarda normalmente en su propio archivo.

10. Props en React

¿Qué son las props en React?
Las props (properties) son datos que un componente recibe desde otro componente. Sirven para enviar información y hacer que un componente sea reutilizable.


Un componente sin props siempre muestra lo mismo. Un componente con props puede mostrar información diferente sin cambiar su estructura.


Regla importante
Las props son de solo lectura. El componente recibe la información y la muestra, pero no debe modificarla.

Ejemplo: Card sin props (problema)

Si creamos varias Cards sin props, todas mostrarán el mismo contenido. Esto no es reutilizable.

11. useState en React

Hasta ahora, nuestros componentes mostraban información fija. El contenido no cambiaba aunque el usuario hiciera clic.

Para que un componente pueda cambiar información en pantalla, React nos da una herramienta llamada useState.

¿Qué es useState?

useState es una función de React que sirve para:

• Guardar un valor dentro del componente.
• Cambiar ese valor cuando ocurre algo.
• Volver a dibujar el componente automáticamente.

Si no usamos useState, React no sabe que algo cambió.

Paso 1: Importar useState

useState no viene activo por defecto. Debemos importarlo desde React.


Componente.jsx

import { useState } from 'react'

Paso 2: Crear un estado

useState se usa dentro del componente.

Cuando lo llamamos, devuelve dos cosas:

• El valor actual.
• Una función para cambiar ese valor.

Ejemplo.jsx

const [numero, setNumero] = useState(0)

Aquí:

• numero es el valor actual.
• setNumero es la función que lo cambia.
• 0 es el valor inicial.

Paso 3: Componente completo con useState

Vamos a crear un componente que muestre un número y lo aumente al hacer clic.


Contador.jsx

import { useState } from 'react'

function Contador() {

  const [numero, setNumero] = useState(0)

  function aumentar() {
    setNumero(numero + 1)
  }

  return (
    <div>
      <p>Valor actual: {numero}</p>

      <button onClick={aumentar}>
        Aumentar
      </button>
    </div>
  )
}

export default Contador

¿Qué está pasando?

• El número empieza en 0.
• Al hacer clic, se ejecuta la función.
• setNumero cambia el valor.
• React vuelve a dibujar el componente.

Regla importante sobre useState

Nunca debes cambiar el estado así: 


numero = numero + 1   ❌

Siempre debes usar la función: 


setNumero(numero + 1) ✅

12. useState + Inputs en React

Hasta ahora aprendimos que useState sirve para guardar información que cambia y se muestra en pantalla.

En esta lección aprenderemos a usar inputs para recibir información del usuario y guardarla usando useState.

¿Qué pasa con un input normal?

En HTML puro, el input guarda el texto solo. React no sabe qué se escribió.

13. useState con booleanos (mostrar y ocultar)

Hasta ahora usamos useState con números y texto.

En esta lección aprenderás a usar useState con valores true y false para mostrar u ocultar elementos en pantalla.

¿Qué es un booleano?

Un booleano es un valor que solo puede ser:

• true (verdadero)
• false (falso)

En React se usan booleanos para decidir si algo se muestra o no en pantalla.

Paso 1: Crear un estado booleano

Vamos a crear un estado que empiece en false. Eso significa que el contenido estará oculto.

Ejemplo.jsx

import { useState } from 'react'

Aquí:

• mostrar guarda true o false.
• setMostrar cambia ese valor.
• false significa oculto.

Paso 2: Componente completo

Vamos a crear un botón que muestre u oculte un texto.

MostrarTexto.jsx

import { useState } from 'react'

function MostrarTexto() {

  const [mostrar, setMostrar] = useState(false)

  function toggle() {
    setMostrar(!mostrar)
  }

  return (
    <div>

      <button onClick={toggle}>
        Mostrar / Ocultar
      </button>

      {mostrar && (
        <p>
          Este texto aparece y desaparece
        </p>
      )}

    </div>
  )
}

export default MostrarTexto

¿Qué significa esta parte del código? 


{mostrar && (
  <p>
    Este texto aparece y desaparece
  </p>
)}

Esta expresión funciona como un if corto dentro del JSX.

Se lee así:

“Si mostrar es true, muestra el párrafo.”
“Si mostrar es false, no muestres nada.”

Es equivalente a escribir: 


if (mostrar) {
  mostrar el <p>
}

React no oculta el elemento con CSS. Simplemente no lo dibuja cuando la condición es falsa.

¿Qué hace esta línea? 


setMostrar(!mostrar)

Esta línea invierte el valor:

• Si es true, pasa a false.
• Si es false, pasa a true.

14. Operador ternario en React (? :)

¿Qué es el operador ternario?

El operador ternario es una forma corta de escribir un if / else.

Tiene esta forma: 


condicion ? opcionSi : opcionNo

Se lee así:

“Si la condición es verdadera, usa la primera opción.”
“Si es falsa, usa la segunda opción.”

Este código:

• Necesitas un if / else

Paso 2: Componente completo

import { useState } from 'react'

function EditarTareas() {

  const [tareas, setTareas] = useState([
    'Estudiar React',
    'Practicar useState'
  ])

  const [textoEditado, setTextoEditado] = useState('')
  const [indiceEditando, setIndiceEditando] = useState(null)

  function iniciarEdicion(index) {
    setIndiceEditando(index)
    setTextoEditado(tareas[index])
  }

  function guardarEdicion() {
    const nuevasTareas = tareas.map((tarea, index) =>
      index === indiceEditando ? textoEditado : tarea
    )

    setTareas(nuevasTareas)
    setIndiceEditando(null)
    setTextoEditado('')
  }

  return (
    <div>

      <h3>Lista de tareas</h3>

      <ul>
        {tareas.map((tarea, index) => (
          <li key={index}>

            {indiceEditando === index ? (
              <>
                <input
                  value={textoEditado}
                  onChange={(e) => setTextoEditado(e.target.value)}
                />
                <button onClick={guardarEdicion}>
                  Guardar
                </button>
              </>
            ) : (
              <>
                {tarea}
                <button onClick={() => iniciarEdicion(index)}>
                  Editar
                </button>
              </>
            )}

          </li>
        ))}
      </ul>

    </div>
  )
}

export default EditarTareas

¿Qué está pasando aquí?

• Se identifica la tarea a editar.
• Se muestra un input solo en esa tarea.
• Se reemplaza el texto al guardar.
• Se crea un array nuevo con map().

[]

23. useEffect con limpieza (cleanup)

En la lección anterior aprendimos a ejecutar código automático usando useEffect.

Ahora veremos cómo detener ese código cuando el componente deja de usarse.

¿Qué es la limpieza (cleanup)?

Algunos efectos crean procesos que siguen activos:

• temporizadores (setInterval, setTimeout)
• escuchas (eventos)
• procesos repetitivos

Cuando el componente desaparece, esos procesos deben detenerse.

Sintaxis de limpieza 


useEffect(() => {

  // código del efecto

  return () => {
    // limpieza
  }

}, [])

La función que se devuelve se ejecuta cuando el componente se desmonta.

Paso 1: Temporizador sin limpieza (problema)

Este ejemplo tiene un problema. 


useEffect(() => {
  setInterval(() => {
    console.log('Ejecutándose...')
  }, 1000)
}, [])

El intervalo nunca se detiene, aunque el componente ya no exista.

Paso 2: Temporizador con limpieza (correcto)

Timer.jsx

import { useEffect, useState } from 'react'

function Timer() {

  const [segundos, setSegundos] = useState(0)

  useEffect(() => {

    const intervalo = setInterval(() => {
      setSegundos(s => s + 1)
    }, 1000)

    return () => {
      clearInterval(intervalo)
    }

  }, [])

  return (
    <div>
      <p>Segundos: {segundos}</p>
    </div>
  )
}

export default Timer

¿Qué está pasando aquí?

• Se crea un intervalo al montar el componente.
• El contador aumenta cada segundo.
• Cuando el componente se elimina, se ejecuta la limpieza.
• El intervalo se detiene con clearInterval.

¿Por qué el array está vacío?

Porque el intervalo debe crearse una sola vez.

La actualización usa esta forma segura: 


setSegundos(s => s + 1)

Así no dependemos del valor externo.

¿Cuándo se ejecuta la limpieza?

• Cuando el componente se desmonta.
• Antes de ejecutar el efecto otra vez (si hay dependencias).

main.jsx

import { useState, useEffect } from 'react'

function UsuariosProvider({ children }) {

  const [usuarios, setUsuarios] = useState([])
  const [cargando, setCargando] = useState(true)
  const [error, setError] = useState(null)

    async function cargarUsuarios() {

        const respuesta = await fetch(
          'https://jsonplaceholder.typicode.com/users'

        setError(err.message)
      } finally {
        setCargando(false)
      }
    }

    cargarUsuarios()

  }, [])

  return (
    <UsuariosContext.Provider
      value={{ usuarios, cargando, error }}
    >
      {children}
    </UsuariosContext.Provider>
  )
}

      const data = await respuesta.json()
      setMensaje('Formulario enviado correctamente')
      console.log('Respuesta:', data)

    } catch (error) {
      setMensaje('Error al enviar el formulario')
    }
  }

  return (
    <form onSubmit={handleSubmit}>

      <input
        name="titulo"
        value={form.titulo}
        onChange={handleChange}
        placeholder="Título"
      />

      <textarea
        name="contenido"
        value={form.contenido}
        onChange={handleChange}
        placeholder="Contenido"
      ></textarea>

      <button type="submit">
        Enviar
      </button>

      {mensaje && (
        <p>{mensaje}</p>
      )}

    </form>
  )
}

import { useState } from 'react'
import { useNavigate } from 'react-router-dom'

function Login() {

  const [username, setUsername] = useState('')
  const [password, setPassword] = useState('')
  const navigate = useNavigate()

  function enviarLogin(e) {
    e.preventDefault()

    fetch('https://dummyjson.com/auth/login', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        username: username,
        password: password
      })
    })
    .then(res => {
      if (!res.ok) {
        throw new Error('Credenciales incorrectas')
      }
      return res.json()
    })
    .then(data => {
      console.log(data)

      // Guardar token
      localStorage.setItem('token', data.token)
      localStorage.setItem('usuario', data.username)

      navigate('/admin')
    })
    .catch(err => {
      alert(err.message)
    })
  }

  return (
    <form onSubmit={enviarLogin}>

      <h2>Iniciar sesión</h2>

      <input
        type="text"
        placeholder="Usuario"
        value={username}
        onChange={e => setUsername(e.target.value)}
      />

      <input
        type="password"
        placeholder="Contraseña"
        value={password}
        onChange={e => setPassword(e.target.value)}
      />

      <button>Ingresar</button>

      <p>
        Usuario de prueba: kminchelle

        Password: 0lelplR
      </p>

    </form>
  )
}

  )
}

function RutaProtegida({ usuario, children }) {

import { useState } from 'react'

function Login({ setUsuario }) {

  const [username, setUsername] = useState('')
  const [password, setPassword] = useState('')
  const [error, setError] = useState('')

  function iniciarSesion(e) {
    e.preventDefault()

    fetch('https://dummyjson.com/auth/login', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json'
      },
      body: JSON.stringify({
        username: username,
        password: password
      })
    })
    .then(res => res.json())
    .then(data => {
      if (data.token) {
        setUsuario(data)
      } else {
        setError('Credenciales incorrectas')
      }
    })
  }

  return (
    <div>
      <h2>Iniciar sesión</h2>

      <form onSubmit={iniciarSesion}>

        <input
          type="text"
          placeholder="Usuario"
          value={username}
          onChange={e => setUsername(e.target.value)}
        />

        <input
          type="password"
          placeholder="Contraseña"
          value={password}
          onChange={e => setPassword(e.target.value)}
        />

        <button>Ingresar</button>

      </form>

      {error && <p>{error}</p>}
    </div>
  )
}

export default Login
